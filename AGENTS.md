# AI Agent 行为规范

## 强制要求

### 语言规范（最高优先级）

**AI 在与用户的所有交互中必须使用中文进行回答，包括：**
- 问题回答
- 代码注释
- 提交信息
- 文档编写
- 错误说明
- 技术解释

**违反此规范的代码修改将被拒绝。**

---

## 工作流程

### 代码修改流程

1. **理解需求** - 确认功能要求和边界条件
2. **创建 Todo** - 分解任务为可执行的原子步骤
3. **执行修改** - 按计划逐步实施
4. **验证结果** - 确保修改符合规范

### 代码质量检查（必选）

代码修改后，必须依次执行：

```bash
pnpm lint        # 静态检查并自动修复
pnpm fmt         # 代码格式化
pnpm build       # 构建验证
```

### 测试要求

- 使用 Vitest 4.0+ 进行单元测试
- 使用 `@testing-library/jest-dom` 进行断言
- 调用 Tauri 命令的代码使用 `@tauri-apps/api/mocks` 模拟后端

---

## 任务委托原则

| 任务类型 | 处理方式 |
|---------|---------|
| 简单文件修改（单文件、已知路径） | 直接执行 |
| 复杂任务（多步骤、多模块） | 创建 Todo 后委托 |
| 外部库/不熟悉的 API | 先咨询 librarian |
| 架构决策/疑难问题 | 先咨询 Oracle |
| UI/动画/样式 | 委托给 visual-engineering |

---

## 错误处理

1. **修复优先** - 发现 bug 时直接修复，不要只是记录
2. **最小修改** - 只改问题所在，避免大范围重构
3. **验证完整** - 修复后必须验证

---

## 禁止事项

- ❌ 使用 `as any`、`@ts-ignore` 抑制类型错误
- ❌ 空 catch 块 `catch(e) {}`
- ❌ 删除测试让测试通过
- ❌ 在未确认的情况下猜测实现
- ❌ 提交未经 lint/build 验证的代码
