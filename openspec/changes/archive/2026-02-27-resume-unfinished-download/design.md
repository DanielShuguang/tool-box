## 上下文

### 背景
当前下载器功能在程序异常退出或网络中断后，无法识别和恢复未完成的下载任务。用户需要手动重新下载整个文件，造成时间和带宽浪费。

### 当前状态
- 前端已有 `DownloadTask` 数据结构，包含 `downloadedBytes`、`etag`、`lastModified` 等断点续传相关字段
- 后端已实现 HTTP Range 请求支持断点续传
- 任务存储在 localStorage，但临时文件格式尚未明确定义

### 约束
- 临时文件需要同时存储下载进度信息供恢复使用
- 服务器必须支持 HTTP Range 才能实现断点续传
- 用户通过手动选择文件方式恢复下载

### 利益相关者
- 下载功能用户：需要恢复中断下载的终端用户
- 开发者：维护下载功能的开发者

## 目标 / 非目标

**目标：**
1. 定义临时文件格式（`.download` + `.download.json`）
2. 实现扫描和识别本地未完成下载文件的功能
3. 实现从断点处恢复下载的功能
4. 提供用户友好的界面选择需要恢复的下载任务
5. 严格验证服务器是否支持断点续传，不支持时给出明确错误提示

**非目标：**
1. 不实现自动扫描目录功能（用户手动选择）
2. 不实现下载任务自动恢复（用户主动触发）
3. 不支持多线程分段下载的断点续传（单线程断点续传）
4. 不处理临时文件加密或压缩

## 决策

### 决策 1：临时文件格式
**选择**：使用 `.download` + `.download.json` 双文件格式

| 方案 | 优点 | 缺点 |
|------|------|------|
| `.download` + `.download.json` | 结构清晰，进度信息独立存储 | 需要维护两个文件 |
| 单文件（带自定义头部） | 只需维护一个文件 | 解析复杂，可能破坏文件完整性 |

**理由**：独立的 JSON 进度文件便于解析和维护，不影响原始文件结构。

### 决策 2：进度文件存储位置
**选择**：与临时文件同目录

**理由**：
- 用户选择临时文件后可直接获取进度文件
- 便于文件管理和清理

### 决策 3：恢复下载时的服务器验证
**选择**：严格验证服务器支持 Range 请求，不支持则报错

| 方案 | 优点 | 缺点 |
|------|------|------|
| 严格验证 | 明确告知用户原因，避免数据损坏 | 需要额外一次 HEAD 请求 |
| 自动降级 | 用户无感知，体验更好 | 可能导致已下载部分损坏 |

**理由**：严格验证可以确保数据完整性，避免因服务器不支持 Range 导致下载失败或数据错乱。

### 决策 4：后端命令设计
**选择**：新增两个独立命令

| 方案 | 优点 | 职责清晰 |
|------|------|----------|
| 两个独立命令 | 单一职责，易于测试 | scan_unfinished_downloads、resume_download |
| 一个命令 + 参数 | 调用简便 | 职责混合 |

**理由**：扫描和恢复是两个独立操作，应该分离。

### 决策 5：前端架构
**选择**：新建 `ScanResumeDialog.vue` 组件 + `useResumeDownload.ts` Hook

**理由**：
- 符合 Vue 3 组合式 API 最佳实践
- 逻辑与视图分离，便于维护
- 与现有 `useDownloadDialog.ts` 模式保持一致

## 风险 / 权衡

### 风险 1：进度文件与临时文件不同步
**描述**：如果临时文件被删除但进度文件仍存在，会导致无效的任务显示

**缓解措施**：
- 恢复时检查两个文件都存在
- 验证进度文件中的文件大小与实际临时文件大小匹配

### 风险 2：服务器不支持 Range
**描述**：部分服务器不支持 HTTP Range 请求

**缓解措施**：
- 严格验证服务器响应头 `Accept-Ranges: bytes`
- 不支持时显示明确错误提示："服务器不支持断点续传，请重新下载"

### 风险 3：文件损坏
**描述**：临时文件可能因磁盘异常等原因损坏

**缓解措施**：
- 恢复时验证临时文件可读
- 校验进度文件 JSON 格式有效性

### 风险 4：URL 失效
**描述**：下载链接可能已过期或文件已删除

**缓解措施**：
- 恢复时先验证 URL 可访问
- 检查 HTTP 响应状态码

### 权衡 1：用户操作复杂度
**描述**：需要用户手动选择文件 vs 自动扫描

**权衡**：手动选择更安全可控，但操作稍繁琐。考虑到数据安全性，选择手动选择更合适。
