## 新增需求

### 需求：临时文件格式定义

系统必须使用 `.download` 作为下载临时文件的后缀，并使用 `.download.json` 作为进度信息文件的后缀。

#### 场景：下载进行时创建临时文件

- **当** 用户开始一个新的下载任务
- **那么** 系统必须在保存目录中创建 `{文件名}.download` 临时文件
- **那么** 系统必须同时创建 `{文件名}.download.json` 进度文件

#### 场景：下载完成时处理临时文件

- **当** 下载任务成功完成
- **那么** 系统必须将临时文件重命名为正式文件名
- **那么** 系统必须删除对应的进度文件

### 需求：进度文件数据结构

进度文件（`.download.json`）必须包含以下字段：

- `url`：原始下载链接
- `downloadedBytes`：已下载字节数
- `totalBytes`：文件总字节数
- `etag`：文件的 ETag 值（可选）
- `lastModified`：文件的最后修改时间（可选）
- `createdAt`：任务创建时间戳
- `startedAt`：任务开始下载时间戳（可选）

#### 场景：读取进度文件

- **当** 用户选择恢复一个未完成的下载任务
- **那么** 系统必须读取并解析 `.download.json` 文件
- **那么** 系统必须验证 JSON 格式有效且包含必要字段

### 需求：扫描未完成下载文件

系统必须提供功能让用户选择本地目录中的 `.download` 临时文件。

#### 场景：用户选择临时文件

- **当** 用户通过文件选择器选择一个 `.download` 文件
- **那么** 系统必须检查关联的 `.download.json` 文件是否存在
- **那么** 系统必须读取进度文件获取下载信息

#### 场景：进度文件不存在

- **当** 用户选择的 `.download` 文件没有对应的 `.download.json` 文件
- **那么** 系统必须显示错误提示："进度文件缺失，无法恢复下载"
- **那么** 该文件必须不出现在可恢复任务列表中

### 需求：服务器断点续传支持验证

系统必须在恢复下载前验证服务器是否支持 HTTP Range 请求。

#### 场景：服务器支持 Range

- **当** 系统向服务器发送 HEAD 请求
- **那么** 如果响应头包含 `Accept-Ranges: bytes`
- **那么** 系统必须使用 Range 请求从断点处继续下载

#### 场景：服务器不支持 Range

- **当** 系统向服务器发送 HEAD 请求
- **那么** 如果响应头不包含 `Accept-Ranges: bytes` 或值不是 `bytes`
- **那么** 系统必须显示确认对话框："服务器不支持断点续传，是否重新下载整个文件？"
- **那么** 如果用户确认重新下载，系统必须删除临时文件并从头开始下载
- **那么** 如果用户取消，系统必须关闭对话框且不执行任何操作

#### 场景：用户确认重新下载

- **当** 服务器不支持 Range 且用户点击"重新下载"
- **那么** 系统必须删除本地临时文件
- **那么** 系统必须删除进度文件
- **那么** 系统必须从头开始下载整个文件

### 需求：恢复下载任务

系统必须能够从已下载的部分继续下载。

#### 场景：成功恢复下载

- **当** 用户确认恢复下载且服务器支持 Range
- **那么** 系统必须发送 HTTP Range 请求，指定 `Range: bytes={downloadedBytes}-`
- **那么** 系统必须将下载内容追加到现有临时文件

#### 场景：文件不匹配

- **当** 系统检测到临时文件大小与记录的 `downloadedBytes` 不一致
- **那么** 系统必须显示错误提示："文件已损坏，无法恢复"
- **那么** 必须将该任务从恢复列表中移除

#### 场景：URL 不可访问

- **当** 系统尝试访问记录的 URL
- **那么** 如果服务器返回 4xx 或 5xx 错误
- **那么** 系统必须显示错误提示："下载链接已失效"

### 需求：用户界面

系统必须提供用户友好的界面让用户选择和恢复未完成的下载任务。

#### 场景：打开恢复对话框

- **当** 用户点击"恢复下载"按钮
- **那么** 系统必须打开文件选择对话框

#### 场景：显示可恢复任务

- **当** 用户选择了一个或多个 `.download` 文件
- **那么** 系统必须显示每个任务的以下信息：文件名、已下载大小、总大小、原始 URL
- **那么** 用户必须能够选择要恢复的任务

#### 场景：批量恢复

- **当** 用户选择了多个未完成的下载任务
- **那么** 系统必须为每个任务创建对应的下载任务
- **那么** 系统必须从各自的断点处开始下载

### 需求：下载过程中的临时文件管理

系统必须在下载过程中正确管理临时文件和进度文件。

#### 场景：定期保存进度

- **当** 下载正在进行时
- **那么** 系统必须定期更新 `.download.json` 文件中的 `downloadedBytes` 值
- **那么** 更新间隔不得超过 5 秒

#### 场景：下载中断

- **当** 下载因网络中断或程序异常而中断
- **那么** 临时文件和进度文件必须保留在磁盘上
- **那么** 用户必须能够通过恢复功能继续下载
