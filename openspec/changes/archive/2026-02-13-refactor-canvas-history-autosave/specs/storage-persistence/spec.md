## 新增需求

### 需求：保存画布数据
系统必须将画布数据保存到 Tauri store 持久化存储中。

#### 场景：成功保存
- **当** 调用保存方法保存画布数据
- **那么** 数据被持久化到 Tauri store 中

#### 场景：保存时出错
- **当** 保存过程中发生错误（如存储不可用）
- **那么** 错误被捕获并记录，不抛出未处理异常

### 需求：加载画布数据
系统必须能够从 Tauri store 中加载之前保存的画布数据。

#### 场景：成功加载
- **当** 调用加载方法获取画布数据
- **那么** 返回保存的画布数据或 null（如果没有保存数据）

#### 场景：加载损坏数据
- **当** 存储中的数据格式损坏，无法解析
- **那么** 返回 null，并清理损坏的数据

### 需求：数据版本管理
系统必须为存储数据添加版本号，支持未来数据格式迁移。

#### 场景：保存数据包含版本
- **当** 保存画布数据
- **那么** 数据中包含 version 字段，值为当前版本号（1）

#### 场景：数据版本兼容
- **当** 加载数据时发现版本号不匹配
- **那么** 根据版本号执行相应的迁移逻辑，或返回 null

### 需求：数据格式验证
系统必须验证存储数据的完整性。

#### 场景：验证必需字段
- **当** 加载数据时
- **那么** 必须验证数据包含 state.json 字段，否则返回 null
