## 新增需求

### 需求：支持插入图片到画板
系统必须允许用户将图片插入到画板中，图片作为画板元素的一部分进行管理。

#### 场景：从本地文件插入图片
- **当** 用户点击工具栏的"插入图片"按钮并选择本地图片文件
- **那么** 系统将图片插入到画板中心位置
- **并且** 图片以默认大小显示
- **并且** 图片可以被选中、移动、缩放和旋转

#### 场景：从剪贴板粘贴图片
- **当** 用户在画板区域执行粘贴操作（Ctrl+V）且剪贴板包含图片
- **那么** 系统将剪贴板中的图片插入到画板中心位置
- **并且** 图片以默认大小显示
- **并且** 图片可以被选中、移动、缩放和旋转

#### 场景：通过拖拽插入图片
- **当** 用户将图片文件拖拽到画板区域并释放
- **那么** 系统将图片插入到拖拽释放的位置
- **并且** 图片以默认大小显示
- **并且** 图片可以被选中、移动、缩放和旋转

### 需求：支持常见图片格式
系统必须支持浏览器原生支持的常见图片格式，包括 PNG、JPEG、WebP、GIF 和 BMP。

#### 场景：插入 PNG 图片
- **当** 用户选择 PNG 格式的图片文件
- **那么** 系统成功插入图片到画板

#### 场景：插入 JPEG 图片
- **当** 用户选择 JPEG 格式的图片文件
- **那么** 系统成功插入图片到画板

#### 场景：插入 WebP 图片
- **当** 用户选择 WebP 格式的图片文件
- **那么** 系统成功插入图片到画板

#### 场景：插入不支持的图片格式
- **当** 用户选择不支持的图片格式文件
- **那么** 系统显示错误提示，告知用户格式不支持

### 需求：图片必须支持移动操作
系统必须允许用户通过拖拽移动画板中的图片。

#### 场景：移动图片
- **当** 用户选中图片并拖拽
- **那么** 图片跟随鼠标移动
- **并且** 释放鼠标后图片停留在新位置

### 需求：图片必须支持缩放操作
系统必须允许用户通过缩放手柄或快捷键调整图片大小。

#### 场景：通过手柄缩放图片
- **当** 用户选中图片并拖拽缩放手柄
- **那么** 图片大小跟随拖拽变化
- **并且** 释放鼠标后图片保持新大小

#### 场景：通过快捷键缩放图片
- **当** 用户选中图片并使用鼠标滚轮
- **那么** 图片以鼠标位置为中心进行缩放
- **并且** 缩放比例在合理范围内（如 10% - 500%）

### 需求：图片必须支持旋转操作
系统必须允许用户通过旋转手柄调整图片角度。

#### 场景：旋转图片
- **当** 用户选中图片并拖拽旋转手柄
- **那么** 图片围绕中心点旋转
- **并且** 释放鼠标后图片保持新角度

### 需求：图片元素必须与画板其他元素统一管理
系统必须将图片元素与画板中的其他图形元素使用相同的数据结构和管理方式。

#### 场景：图片元素数据结构
- **当** 图片插入到画板
- **那么** 图片元素包含类型标识、位置、大小、旋转角度、图片数据等属性
- **并且** 图片元素可以被序列化和反序列化

#### 场景：图片元素与其他元素共存
- **当** 画板中同时存在图片和图形元素
- **那么** 所有元素统一存储在画板元素列表中
- **并且** 所有元素可以被统一选中、删除、复制

### 需求：图片插入失败时必须显示错误提示
系统必须在图片插入失败时显示友好的错误提示信息。

#### 场景：文件读取失败
- **当** 用户选择的图片文件读取失败
- **那么** 系统显示错误提示，告知用户文件读取失败

#### 场景：图片格式不支持
- **当** 用户选择的图片格式不被支持
- **那么** 系统显示错误提示，告知用户格式不支持

#### 场景：图片过大
- **当** 用户选择的图片尺寸超过限制
- **那么** 系统显示警告提示，询问用户是否继续插入
