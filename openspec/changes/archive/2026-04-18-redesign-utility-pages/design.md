## 上下文

现有工具页面需要统一升级为新的设计语言，参考已改造的待办、记账等页面的设计风格。

**当前状态：**
- 6个工具页面使用旧的设计
- 缺乏统一的视觉风格
- 交互体验不一致

**约束：**
- 继续使用 Naive UI 组件库
- 使用 UnoCSS 进行样式管理
- 保持组件结构简洁（单文件不超过300行）
- 不引入新的外部依赖

## 目标 / 非目标

**目标：**
- 统一 6 个工具页面的视觉风格
- 应用新的设计语言（卡片、统计面板、hover 反馈）
- 改善交互体验

**非目标：**
- 不修改业务逻辑
- 不添加新功能
- 不引入新的依赖库
- 不重构后端代码

## 决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 布局模式 | 统计卡片 + 主内容区 | 参照 Todo 页面，提供直观的数据概览 |
| 容器样式 | 圆角卡片 + 轻阴影 | Naive UI 主题变量 + CSS 自定义 |
| 操作反馈 | hover 显示按钮 | 减少界面复杂度，保持简洁 |
| 状态指示 | 颜色编码 | 绿色/橙色/红色表示不同状态 |
| 空状态 | 图标 + 引导文案 | 提供清晰的用户反馈 |

## 风险 / 权衡

- **与现有功能冲突** → 通过逐页改造降低风险，每页独立测试
- **样式变量命名不一致** → 统一使用 Naive UI theme vars (camelCase)
- **响应式布局** → 使用 UnoCSS 响应式工具类 (sm:, lg:)
